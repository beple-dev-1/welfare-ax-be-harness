# 자주 놓치는 토픽 리마인더 (WHAT 영역)

`dev-interview` 자율 선탐색 통합 단계 + code-investigator·db-meta-manager·security-auditor 3 sub-agent 모두 동일 파일 참조.

기획서·코드·DB 어디에서도 확인되지 않으면 **갭으로 등록**한다.

---

## 8 항목 체크리스트

| # | 토픽 | 검증 포인트 |
|---|------|------------|
| 1 | **외부 통신 보안 스펙** | HMAC · SEED · 서명 · 암호화 (브리프 §6 핵심) |
| 2 | **PII / 개인정보 처리** | 마스킹 · 암호화 · 로그 보호 |
| 3 | **권한·역할** | 메뉴·기능 접근 권한 (대상자별 분기 포함) |
| 4 | **트랜잭션 경계** | 외부 API 실패 시 롤백 범위 |
| 5 | **알림 연계** | 알림톡 · 이메일 · 푸시 (가입승인·지급완료 등 패턴) |
| 6 | **에러 처리·응답 코드** | ResponseCode 신규/기존 매핑 |
| 7 | **예외 시나리오** | 동시 접속·중복 신청·타임아웃·재처리 |
| 8 | **운영 정책** | Excel 다운로드 한도·세션 만료·점검 시간 |

## HOW 영역 (이 인터뷰에서 묻지 않음)

다음은 `dev-plan` 영역으로 이관 — dev-interview 갭으로 등록 금지:

- API 버전 호환성
- 캐시 TTL
- 배치 볼륨
- 전문 규격 세부

> WHAT (무엇을 만들 것인가) ≠ HOW (어떻게 만들 것인가).
