# Phase 0-1 기본 정보 수집 체크리스트

> dev-interview SKILL.md Phase 0-1 단계 진입 시 lazy Read.
> 5항목을 **1개씩 순서대로** 1:1 질문. 사용자가 한 번에 여러 정보 제공하면 받은 항목 skip.

---

## 질문 항목

| 순서 | 항목 | 용도 | 전달 방식 |
|------|------|------|------|
| 1 | **개발 대상 프로젝트** | 자동 매칭 대조 | `AskUserQuestion` — options = `{{config.projects[].name}}` |
| 2 | **과업번호** | `taskId` (출력 파일명) | 텍스트 (예: 057, 1234) |
| 3 | **작업 브랜치** | 브리프 §1 메타 | 텍스트 (부모+feature. 미정이면 "미정") |
| 4 | **개요** | 인터뷰 출발점 | 텍스트 (한두 문장) |
| 5 | **기획서 파일 경로** | Phase 0-2 입력 | 텍스트 (없으면 "없음") |

---

## 모드별 사전 보유

| 모드 | 사전 보유 항목 | 질문 항목 |
|------|---------------|----------|
| 기획서 파일 | 5번 | 1~4 |
| 주제 | 4번 (일부) | 1~3 후 4를 "주제 외 추가 설명?"으로 확장 |
| 대화형 | (없음) | 1~5 전체 |

## 질문 형식

**항목 1 (프로젝트)** — `AskUserQuestion` 1질문 호출:

```jsonc
{
  "questions": [{
    "header": "프로젝트",
    "question": "개발 대상 프로젝트가 어디인가요? (project.yaml 인벤토리)",
    "multiSelect": false,
    "options": [
      { "label": "{projects[0].name}", "description": "{shortName / role 1줄}" },
      { "label": "{projects[1].name}", "description": "..." }
      // ... 최대 4개
    ]
  }]
}
```

> - options = `{{config.projects[].name}}`. 인벤토리가 4개 초과면 도구 4옵션 제한에 맞춰 **빈도·맥락상 후보 우선 노출** + 나머지는 자동 "Other" 선택지로 흡수.
> - 사용자가 Other 선택 시 자유 입력값을 프로젝트명으로 사용 (다시 묻지 않음).

**항목 2~5** — 텍스트 1:1 질문 (선택지 없음). 근거 1줄 제시 후 질문.

---

## 종료 후 자동 인계

5항목 답변 누적 완료 시 **사용자 확인 없이** 다음 단계로 즉시 인계:

- 기획서 파일 있음 (5번 ≠ "없음") → Phase 0-2 parse-spec-doc Skill 호출. parse 완료 후 자율 선탐색 자동 트리거.
- 기획서 파일 없음 → Phase 0-2 skip. 자율 선탐색 즉시 자동 트리거.

> "탐색 시작해도 될까요?" / "다음 단계로 갈까요?" 컨펌 질문 금지. 5항목 답변 자체가 트리거.
