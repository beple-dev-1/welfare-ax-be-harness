# Q&A 로그 부록 템플릿

`dev-interview` 가 11섹션 본문 + 메타 footer 사이에 **자동 첨부**하는 마크다운 블록.

---

## 부록 블록 (자동 첨부)

```markdown
## 인터뷰 Q&A 로그 (부록)

> dev-plan 입력 스키마(§1-§11)에 영향 없는 메타데이터.
> **사용자 원문 인용 금지** — 결정 사실만 1줄.

| # | 카테고리 | 강도 | 질문 요약 (≤80자) | 응답 요약 (≤80자) | 반영 |
|---|---------|------|-------------------|-------------------|------|
| 1 | B. 설계 결정 | STRONG | SSO 토큰 검증 위치 (Filter vs Interceptor) | Interceptor 통일 | §6-1, §3-3 |
| 2 | A. 외부 스펙 | MEDIUM | HMAC 키 교체 주기 별첨 존재? | 사업부 확인 필요 → 보류 | §9 |
| 3 | C. 코드값 미정 | STRONG | 신청 상태값 추가 (REQ/APPR/REJ) | DB 신규 코드값 등록 | §5-1, §3-4 |
| 4 | D. 기획서 모호 | WEAK | 마감일 한도 명세? | 운영 기준으로 위임 → §9 보류 | §9 |
```

---

## 컬럼 정의

| 필드 | 값 형식 | 비고 |
|------|---------|------|
| `#` | 라운드 번호 (1부터) | 재인터뷰 라운드도 연속 부여 |
| `카테고리` | `A. 외부 스펙` / `B. 설계 결정` / `C. 코드값 미정` / `D. 기획서 모호` | 4개 중 택1 |
| `강도` | `STRONG` / `MEDIUM` / `WEAK` | 라운드 헤더의 강도와 일치 |
| `질문 요약` | ≤80자 | 헤더의 질문을 1줄 요약 (옵션 표시 생략) |
| `응답 요약` | ≤80자, 결정 사실만 | **사용자 원문 인용 금지**. 예: "Interceptor 통일" (O) vs "사용자가 'Interceptor로 하자고 함'" (X) |
| `반영` | `§X-Y` 형식 또는 `§9` (보류) | 재인터뷰 라운드는 `§X (재인터뷰 R{n})` 표기 |

---

## 누적 규칙 (dev-interview SKILL.md 라운드 종료 시점)

1. 각 1:1 라운드 종료 직후 (사용자 응답 수신 후) 내부 변수에 push.
2. 재인터뷰 라운드 (Stage 2 RED 후) 도 동일 누적, "반영" 컬럼에 `(재인터뷰 R{n})` 라벨 추가.
3. 사용자가 `AskUserQuestion` 의 **Other 선택 또는 자유 입력** → 해당 라운드 응답 요약은 "객관식 거부 — 다음 라운드 재질의" 로 기록.
4. 사용자가 "모르겠음" / "알아서 해줘" (또는 `추천대로(잘 모르겠음)` 옵션) → 응답 요약은 "기본값 적용: {기본값}" 형식.

---

## 보안 규칙

- **사용자 원문 인용 절대 금지** (PII 노출 방지).
- 응답 요약은 항상 **결정 사실** 또는 **합의된 행동**으로 paraphrase.
- 갭 보류 시 "→ §9 보류" 라벨로 통일.

---

## 미사용 (Q&A 라운드 0회) 시

인터뷰 도중 1:1 라운드가 1회도 발생하지 않은 경우 (예: 기획서 + 코드 + DB 만으로 모든 갭 해소) 부록 블록 자체를 생략한다. 빈 표를 만들지 않는다.
