# 커밋 메시지 규칙

> `/git commit` 서브커맨드에서 참조하는 커밋 메시지 컨벤션.
> base-rule.md §7 「Git Commit 메시지 컨벤션」에서 추출.

---

## 커밋 타입

| 타입 | 형식 | 자동 감지 조건 |
|------|------|---------------|
| `ADD` | `ADD : {메시지}` | 신규 파일(A)만 있을 때 |
| `CHANGE` | `CHANGE : {메시지}` | 수정된 파일(M)이 있을 때 |
| `DELETE` | `DELETE : {메시지}` | 삭제된 파일(D)만 있을 때 |
| `REVERT` | `REVERT : {메시지}` | 사용자 지정만 가능 |
| `MERGE` | `MERGE : {메시지}` | 사용자 지정만 가능 |

---

## 자동 감지 알고리즘

`git diff --staged --name-status` 결과를 분석한다:

1. **A(Added)만 존재** → `ADD`
2. **D(Deleted)만 존재** → `DELETE`
3. **M(Modified) 포함 또는 A+D 혼합** → `CHANGE`
4. **R(Rename) 포함** → `CHANGE` (파일명 변경은 수정으로 분류)
5. **사용자가 명시적으로 타입 지정** → 해당 타입 사용

---

## 최종 형식

```
{TYPE} : {메시지} (#{현재브랜치})
```

> **HEREDOC 사용:** 커밋 메시지에 특수 문자(따옴표, `$` 등)가 포함될 수 있으므로 HEREDOC 형식으로 커밋한다:
> ```bash
> git commit -m "$(cat <<'EOF'
> {TYPE} : {메시지} (#{현재브랜치})
> EOF
> )"
> ```

### 예시

```
ADD : 회원가입 API 구현 (#feature/1001)
```

```
CHANGE : 로그인 토큰 만료 시간 수정 (#feature/1002)
```

```
DELETE : 미사용 레거시 UserHelper 클래스 제거 (#feature/1003)
```

```
REVERT : 배포 오류로 인한 이전 커밋 되돌리기 (#hotfix/2001)
```

```
MERGE : feature/1001 → develop 병합
```

## 커밋 메시지 작성 규칙

1. **한글로 작성**한다
2. **동사형 종결** (예: "구현", "수정", "제거", "추가")
3. **50자 이내** 권장 (브랜치 태그 제외)
4. **what(무엇)보다 why(왜)** 중심으로 작성
5. 브랜치 태그 `(#{branch})`는 자동 추가되므로 사용자가 입력하지 않음
