# HANDOFF.md 템플릿

> pack 스킬 **4단계** 에서 참조하는 워크스페이스 루트 `HANDOFF.md` 작성 형식.
> 이 파일은 **단일 파일** 이며 모든 프로젝트의 진행중 컨텍스트를 프로젝트별 섹션으로 통합한다. **`### Done` 섹션은 절대 두지 않는다** (HISTORY 가 누적 책임).

<Frontmatter>

```yaml
---
projects:
  - {프로젝트명1}: {브랜치명1}
  - {프로젝트명2}: {브랜치명2}
updated: {ISO 8601 KST, 초 생략 — 예: 2026-04-25T14:30+09:00}
---
```

- `projects:` 의 각 항목은 작성 시점의 git 브랜치를 기록한다.
- detached HEAD 시 `{프로젝트명}: _detached_{short-sha}` 형식.
- 다음 pack 호출 시 0단계 P2 가 이 frontmatter 를 읽어 stale 여부를 판정한다.

</Frontmatter>

<Project_Section>

각 프로젝트마다 아래 구조의 섹션을 작성한다. 진행중 컨텍스트(Plan/Next/Caution)가 있는 프로젝트만 섹션을 만든다 — 단순 변경(메타 작업 등)만 있고 진행중 컨텍스트가 없으면 섹션을 만들지 않고 변경 파일은 다른 프로젝트의 Files 또는 `Common Files` 로 흡수한다.

```markdown
## {프로젝트명} @ `{브랜치명}`

### Plan

> plan 모드에서 수립한 계획이 없으면 이 섹션은 생략한다.

- **목표**: {계획의 최종 목표}
- **접근 방식**: {합의한 구현 전략 요약}
- **주요 결정**: {선택한 방안과 그 이유}
- **진행 상태**: {계획 대비 현재 진척도 — 예: "5단계 중 3단계 완료"}

### Next
1. {다음에 할 작업 1}
2. {다음에 할 작업 2}

### Caution
- {주의사항 1}
- {주의사항 2}

### Files
- `{경로}` — {역할/변경 내용 요약} [커밋됨|미커밋|미커밋-신규]

---
```

**갱신 규칙** (4단계에서 적용):

1. 이번 세션에서 변경된 프로젝트의 섹션만 갱신.
2. **Plan 갱신**: 이번 세션에서 plan 모드를 사용했으면 교체. 사용하지 않았으면 이전 Plan 유지하되 `진행 상태` 만 이번 Done 기반으로 업데이트.
3. **Next, Caution**: 이번 세션 기준으로 갱신. 이전 Next 중 완료된 항목은 자연스럽게 빠지고, 그 항목은 이번 HISTORY entry 의 Done 으로 들어간다.
4. **Files**: 병합. 동일 파일은 최신 변경 내용으로 갱신.
5. 이번 세션에 손대지 않은 다른 프로젝트의 섹션은 **그대로 유지** — 다른 진행 작업이 있으면 빠지면 안 된다.
6. **`### Done` 섹션이 남아있으면 제거** — Done 은 HISTORY 의 책임이며 HANDOFF.md 에는 두지 않는다.

</Project_Section>

<Cross_Cutting>

```markdown
## Cross-Cutting Caution

> 워크스페이스 전체 또는 **둘 이상의 프로젝트에 걸친 인터페이스/의존성**에서만 의미 있는 항목만. 단일 프로젝트 한정은 그쪽 `Caution` 에만.

- [{프로젝트A} ↔ {프로젝트B}] {프로젝트 간 의존성·인터페이스 주의}
- [공통] {워크스페이스 메타 또는 다수 프로젝트 공통 주의}
```

**정합성 점검** (4단계에서 매번):

- 단일 프로젝트 한정 항목이 Cross-Cutting 에 섞여 있으면 해당 프로젝트의 `Caution` 으로 이동시키고 Cross-Cutting 에서 제거.
- 인라인 태그(`[프로젝트A ↔ 프로젝트B]`, `[공통]`)는 항상 표기.

</Cross_Cutting>

<Common_Files>

```markdown
## Common Files

> `.claude/`, `~/.claude/`, 워크스페이스 루트 직접 등 **어떤 프로젝트에도 속하지 않는** 파일만.

- `.claude/{경로}` — {역할/변경 내용 요약} [커밋됨|미커밋|미커밋-신규]
```

**등록 기준**:

- 프로젝트 디렉토리(`{프로젝트명}/` 등) 하위 파일은 절대 `Common Files` 에 두지 않는다.
- 어떤 프로젝트에도 속하지 않는 파일만 등록 — 워크스페이스 루트 직접(`HANDOFF.md` 자기 자신, `HANDOFF_HISTORY.md` 등은 제외하고 메타 파일만), `.claude/` 하위, `target/` 산출물 등.

</Common_Files>

<Commit_Status_Tag>

`Files` / `Common Files` 의 각 파일에 커밋 상태 태그를 표기한다:

| 상태 | 표시 | 의미 |
| ---- | ---- | ---- |
| 커밋 완료 | `[커밋됨]` | git 에 커밋된 상태 |
| 미커밋 (tracked) | `[미커밋]` | 변경되었으나 아직 커밋되지 않음 |
| 미커밋 (untracked) | `[미커밋-신규]` | 새로 생성된 파일, git 에 추가되지 않음 |

`git status --short` 출력으로 판정.

</Commit_Status_Tag>
